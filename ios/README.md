# BOR Systems — iOS app

Native SwiftUI app for iPhone and iPad. Targets **iOS 16.0+**, which covers every iPhone from the **iPhone 8** onwards (including SE 2nd/3rd gen, iPhone X/XS/XR/11/12/13/14/15/16 in all sizes).

## Open and run

```sh
cd ios
./setup.sh                      # generates BORSystems.xcodeproj via XcodeGen
open BORSystems.xcodeproj       # opens in Xcode
```

In Xcode: pick any iPhone simulator from the device dropdown and press **Run** (⌘R).

## Connecting to the backend

`BORSystems/AppConfig.swift` controls the API base URL:

```swift
static let apiBaseURL = URL(string: "http://localhost:3000")!
```

- **Simulator** can talk to `localhost:3000` directly (your Mac is the host).
- **Real device on the same WiFi**: change to `http://YOUR-MAC-LAN-IP:3000` (e.g. `http://192.168.1.42:3000`). Find your IP with `ipconfig getifaddr en0`.
- **Production**: change to your deployed `https://` URL.

## What's in the app today

- **Login** with admin-issued credentials (Keychain-stored token).
- **Active alerts** list with auto-refresh every 5 s, pulls from `/alerts/active`.
- **Alert detail** with **I'm on it**, **Sign damaged**, **Sign missing**, manual close + optional note.
- **Dispatches** section on the home screen — tap **On my way** / **Mark done**.
- **On/off-duty** toggle in the toolbar.
- **Push registration** — registers an APNs device token with the backend (real delivery requires Firebase Messaging or APNs HTTP/2 wiring; the token is stored, ready for either).
- **Profile sheet** with sign-out.

## Project layout

```
ios/
├── project.yml              XcodeGen config — single source of truth
├── setup.sh                 ./setup.sh regenerates BORSystems.xcodeproj
├── BORSystems/
│   ├── BORSystemsApp.swift  App entry, AppDelegate for APNs
│   ├── ContentView.swift    Auth router (Login vs Home)
│   ├── LoginView.swift
│   ├── HomeView.swift       Alerts + Dispatches list
│   ├── AlertDetailView.swift
│   ├── ProfileView.swift
│   ├── AppConfig.swift      Backend URL
│   ├── APIClient.swift      Async/await URLSession wrapper
│   ├── AuthStore.swift      ObservableObject for auth state
│   ├── Keychain.swift       Token storage
│   ├── Models.swift         Codable types matching the backend
│   ├── Info.plist           Generated by XcodeGen
│   └── Assets.xcassets/
└── BORSystems.xcodeproj/    Generated — do not edit by hand
```

The `.xcodeproj` is regenerated by `setup.sh` from `project.yml`. Edit `project.yml` to add/remove targets, change the deployment target, etc. — never edit the `.pbxproj` directly.

## Pushing to a real device

Open the project in Xcode, select the **BORSystems** target, **Signing & Capabilities**, and pick your Apple ID team. A free Apple ID works for personal device installs (7-day expiry); a paid Apple Developer account is needed for App Store distribution.
